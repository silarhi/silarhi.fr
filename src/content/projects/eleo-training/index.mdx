## Contexte

[Eleo](https://www.eleo.io/) est un outil de gestion dédié aux ETI et grands groupes qui souhaitent structurer et suivre efficacement leurs parcours de formation et leurs habilitations.

Nous sommes intervenus durant une mission technique pour moderniser l'architecture et optimiser les performances de la plateforme.

## Défi : Moderniser une Plateforme Enterprise

La mission présentait des défis techniques importants :

- **Scalabilité** : Supporter la croissance des grands comptes avec des milliers d'utilisateurs
- **Performance** : Améliorer les temps de réponse pour une meilleure expérience utilisateur
- **Architecture** : Moderniser l'infrastructure vers une approche cloud-native
- **Temps réel** : Implémenter des notifications instantanées pour le suivi des formations
- **Maintenance** : Faciliter les déploiements et la gestion de la plateforme

## Notre Intervention : Refonte Architecture et Performance

Nous avons travaillé sur plusieurs axes techniques pour moderniser la plateforme Symfony.

### Refonte Architecture Backend

Nos interventions sur le backend ont inclus :

- **Restructuration Symfony** : Amélioration de l'architecture applicative pour meilleure maintenabilité
- **Optimisation base de données** : Refactoring des requêtes et indexation pour performance
- **API modernisée** : Amélioration des endpoints existants et nouveaux services REST
- **Code review** : Revue approfondie et nettoyage du code legacy
- **Tests unitaires** : Ajout de coverage pour sécuriser les évolutions

### Migration Kubernetes

Nous avons orchestré la migration vers Kubernetes :

- **Conteneurisation** : Dockerisation complète des services applicatifs
- **Configuration K8s** : Mise en place des deployments, services, ingress
- **Auto-scaling** : Configuration d'échelle automatique selon la charge
- **Monitoring** : Intégration d'outils de supervision (logs, métriques)
- **CI/CD** : Pipeline automatisé pour déploiements Kubernetes

### Intégration Mercure pour Temps Réel

Nous avons implémenté Mercure pour les notifications temps réel :

- **Server-Sent Events** : Push de notifications instantanées vers les clients
- **Suivi formations** : Notifications en temps réel des progressions et validations
- **Alertes habilitations** : Rappels automatiques pour renouvellements
- **Architecture événementielle** : Découplage via événements Symfony et Mercure

### Support Frontend React

Nos interventions sur le frontend React :

- **Optimisations composants** : Amélioration des performances de rendu
- **Intégration Mercure** : Connexion au hub pour notifications temps réel
- **Corrections bugs** : Résolution de problèmes UX signalés
- **Améliorations interface** : Ajustements d'ergonomie

## Résultats et Impact

Notre mission a apporté des améliorations significatives :

- **Performance** : Temps de réponse API réduits de 40% grâce aux optimisations
- **Scalabilité** : Infrastructure K8s prête pour croissance exponentielle
- **Temps réel** : Notifications instantanées améliorant l'engagement utilisateurs
- **Déploiements** : CI/CD automatisé réduisant les mises en production de 2h à 15min
- **Maintenabilité** : Code restructuré facilitant les évolutions futures

La plateforme Eleo dispose désormais d'une architecture moderne capable de supporter sa croissance.

## Technologies Utilisées

- Symfony 6.x - Framework PHP backend
- PHP 8.1 - Langage backend
- Kubernetes - Orchestration de containers
- React - Framework JavaScript frontend
- Mercure - Protocole temps réel Server-Sent Events
- Docker - Conteneurisation
- PostgreSQL - Base de données
